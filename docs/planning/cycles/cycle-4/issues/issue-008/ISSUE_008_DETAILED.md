# C4-008: Remove Test Snapshots and Exclude from Git

## Issue Metadata

| Attribute       | Value              |
| --------------- | ------------------ |
| Issue ID        | C4-008             |
| Area            | REPO               |
| Difficulty      | Trivial            |
| Labels          | contracts, trivial |
| Dependencies    | None               |
| Estimated Lines | 5-10               |

## Overview

Twenty-four JSON snapshot files are currently tracked in the repository under `apps/contracts/contracts/defi-rwa/test_snapshots/test/`. These files are generated by the Soroban test suite and must not be committed.

## Removal Steps

### Step 1: Remove tracked files from git

From the repository root:

```bash
git rm -r apps/contracts/contracts/defi-rwa/test_snapshots/
```

This stages the deletion. The files will be removed from the repository history on the next commit but the directory will remain locally if the tests regenerate it.

### Step 2: Update contracts .gitignore

Open `apps/contracts/.gitignore` and add:

```
# Soroban test snapshots
test_snapshots/
```

### Step 3: Update root .gitignore

Open the root `.gitignore` and add a general entry to catch snapshot files from any tool:

```
# Test snapshots
*.snap
**/test_snapshots/
```

### Step 4: Verify tests still pass

The snapshot files are used by the Soroban test runner to compare expected contract state. After removing them, run:

```bash
cd apps/contracts
cargo test
```

The test runner will regenerate the snapshot files on first run. These new files will be in the working directory but will not be tracked by git because of the `.gitignore` entry. Confirm that all tests pass.

### Step 5: Confirm git ignores the directory

After running the tests, verify the snapshot directory is not staged:

```bash
git status
```

The `test_snapshots/` directory should not appear in the output. If it does, the `.gitignore` entry was not applied correctly.

## Definition of Done

- `git rm` has removed all 24 snapshot files from the index.
- `test_snapshots/` is in `apps/contracts/.gitignore`.
- `*.snap` and `**/test_snapshots/` are in the root `.gitignore`.
- `cargo test` passes after the removal.
- The snapshot directory is not tracked after tests regenerate it.
- All CI workflows pass on the pull request.
